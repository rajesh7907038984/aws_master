Step-by-Step Setup Guide

1
Create Azure AD App Registration
  1. Go to Azure Portal
  2. Navigate to Azure Active Directory → App registrations
  3. Click "New registration"
  4. Enter a name (e.g., "LMS Teams Integration")
  5. Select account type: "Accounts in this organizational directory only"
  6. Leave Redirect URI blank for now
  7. Click "Register"
2
Copy Required Credentials

From your app registration Overview page:

  • Application (client) ID - Copy this as your Client ID
  • Directory (tenant) ID - Copy this as your Tenant ID
3
Create Client Secret
  1. In your app registration, go to "Certificates & secrets"
  2. Click "New client secret"
  3. Add a description (e.g., "LMS Integration Secret")
  4. Select expiration period (recommended: 24 months)
  5. Click "Add"
  6. ⚠️ Important: Copy the Value immediately - you won't be able to see it again!
  7. Save this as your Client Secret
4
Configure API Permissions
  1. Go to "API permissions" in your app registration
  2. Click "Add a permission" → "Microsoft Graph"
  3. Select "Application permissions"
  4. Add the following permissions:

Required Permissions:

  • Calendars.ReadWrite
  • Group.Read.All
  • OnlineMeetings.ReadWrite
  • User.Read.All

5. Click "Add permissions"

5
Grant Admin Consent
  1. In the "API permissions" page, locate the button "Grant admin consent for [Your Organization]"
  2. Click this button (requires Global Administrator or Privileged Role Administrator)
  3. Confirm by clicking "Yes"
  4. Verify all permissions show a green checkmark under "Status"

Note: Admin consent is required for application-level permissions to work properly.

6
Configure Integration in LMS
  1. Click the "Add Teams Integration" button below
  2. Enter a descriptive name for your integration
  3. Paste the Client ID from Step 2
  4. Paste the Client Secret from Step 3
  5. Paste the Tenant ID from Step 2
  6. Click "Add Integration"
  7. Test the connection using the "Test Connection" button
Need Help?
  • View Microsoft's official documentation for app registration
  • Ensure you have Global Administrator or Application Administrator role in Azure AD
  • Client secrets expire - set a calendar reminder to renew before expiration
  • Contact your system administrator if you encounter permission issues

Azure AD Configuration

Step-by-Step Azure AD Setup:
1. Register Application:
  • Go to Azure Portal → Azure Active Directory → App registrations
  • Click "New registration"
  • Name: {{ request.user.branch.name|default:"Your Company" }} LMS SharePoint Integration
  • Supported account types: "Accounts in this organizational directory only"
  • Redirect URI: Web → {{ request.scheme }}://{{ request.get_host }}/account/sharepoint/callback/
2. Configure API Permissions:
  • Go to API permissions → Add permission → Microsoft Graph → Application permissions
  • Required permissions:
    • Sites.ReadWrite.All - Read and write to SharePoint sites
    • User.Read.All - Read all user profiles
    • Files.ReadWrite.All - Read and write files in all site collections
    • Directory.Read.All - Read directory data
  • Important: Click "Grant admin consent for [Your Organization]" after adding permissions
3. Create Client Secret:
  • Go to Certificates & secrets → Client secrets → New client secret
  • Description: LMS SharePoint Integration Secret
  • Expires: Select "24 months" (recommended for production)
  • Click "Add" → Immediately copy the "Value" (not the Secret ID)
  • 🚨 CRITICAL: Save the secret value NOW - it disappears forever after you leave this page!
What to copy:
• Value: Long string starting with something like "8Q~3v2..."
• Secret ID: UUID-like "a1b2c3d4-e5f6..."

Found in Azure AD → Properties → Tenant ID (format: 12345678-1234-1234-1234-123456789abc)

Found in Azure AD → App Registrations → Your App → Application (client) ID (format: 87654321-4321-4321-4321-abcdef123456)

Generated in Azure AD → App Registrations → Your App → Certificates & secrets (starts with characters like "8Q~3v2...")

Example: https://acmecorp.sharepoint.com/sites/lms or https://contoso.sharepoint.com/sites/training
