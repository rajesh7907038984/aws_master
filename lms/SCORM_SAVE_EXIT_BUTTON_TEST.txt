SCORM Save & Exit Button - Testing Guide
==========================================
Date: October 12, 2025

ISSUE: Save & Exit button not working on https://staging.nexsy.io/scorm/view/97

DIAGNOSTIC STEPS:
-----------------

1. **Open the SCORM Player**
   - Go to: https://staging.nexsy.io/scorm/view/97
   - Login if needed
   - SCORM content should load

2. **Open Browser Console** (F12 or Ctrl+Shift+I)
   - Switch to "Console" tab
   - Keep it open while testing

3. **Check Initial State**
   Run in console:
   ```javascript
   console.log('Attempt ID:', ATTEMPT_ID);
   console.log('API Initialized:', window.API && window.API._initialized);
   console.log('API Endpoint:', API_ENDPOINT);
   ```
   
   Expected output:
   - Attempt ID: (a number, not 'None' or undefined)
   - API Initialized: true
   - API Endpoint: /scorm/api/[number]/

4. **Test Save & Exit Button**
   - Click the "Save & Exit" button at the top
   - Watch the console for log messages

   Expected console logs:
   ```
   💾 Save & Exit button clicked - saving progress...
   ⏱️ Setting session_time from client-side tracker: [time]
   💾 Committing SCORM bookmark data for resume...
   ✅ SCORM Commit successful
   ✅ Backend sync successful
   ✅ All tracking data saved to database
   ```

   Expected behavior:
   - Button text changes to "Saving progress..."
   - After 0.5-1 second: "Saved! Redirecting..."
   - Redirects to topic view page

5. **If Button Doesn't Work - Check These:**

   A. **Check if button exists**:
   ```javascript
   document.getElementById('backToTopicBtn')
   ```
   Should return: <a href="#" id="backToTopicBtn"...>

   B. **Check if click handler is attached**:
   ```javascript
   document.getElementById('backToTopicBtn').onclick
   ```
   Should return: function handleBackToTopic(event) {...}

   C. **Manually trigger the function**:
   ```javascript
   handleBackToTopic({preventDefault: function(){}})
   ```
   Should save and redirect

   D. **Check for JavaScript errors**:
   Look for red error messages in console

   E. **Check API initialization**:
   ```javascript
   window.API._initialized
   window.API_1484_11
   ```

6. **Common Issues & Fixes**:

   Issue 1: "Attempt ID not found"
   - Cause: Session not initialized
   - Fix: Refresh page, clear cache

   Issue 2: "API not initialized"
   - Cause: SCORM API failed to load
   - Fix: Check network tab for failed API calls

   Issue 3: "Emergency fallback timeout"
   - Cause: Backend API not responding
   - Fix: Check server logs, verify API endpoint is accessible

   Issue 4: Button does nothing
   - Cause: Click handler not attached
   - Fix: Check if JavaScript loaded correctly

7. **Backend API Test**:

   Test if the backend API is responding:
   ```javascript
   fetch('/scorm/api/' + ATTEMPT_ID + '/', {
       method: 'POST',
       headers: {
           'Content-Type': 'application/json',
           'X-CSRFToken': document.querySelector('[name=csrfmiddlewaretoken]').value
       },
       body: JSON.stringify({
           method: 'Commit',
           parameters: ['']
       })
   })
   .then(r => r.json())
   .then(d => console.log('API Response:', d))
   ```

   Expected: {success: true, result: "true"}

8. **Check Server Logs**:

   On server:
   ```bash
   tail -f /home/ec2-user/lmslogs/gunicorn_error.log
   ```

   Look for:
   - SCORM API errors
   - 500 errors
   - Database errors

KNOWN WORKING CONFIGURATION:
-----------------------------
- Launch URL: scormcontent/index.html (NOT scormdriver/indexAPI.html)
- SCORM Version: 1.2
- API: Synchronous XHR (not async)
- Button ID: backToTopicBtn
- Function: handleBackToTopic()

RECENT FIXES APPLIED:
---------------------
1. ✅ Fixed [object Promise] issue - converted to synchronous API
2. ✅ Fixed launch URL priority - using scormcontent/ instead of scormdriver/
3. ✅ Fixed all 19 packages with wrong launch URLs
4. ✅ Server restarted with new configuration

IF BUTTON STILL NOT WORKING:
-----------------------------

Check these specific issues:

1. **Cross-Origin Issue**:
   - SCORM content may be loading from different origin
   - Check iframe src in Elements tab
   - Should be: /scorm/content/97/scormcontent/index.html

2. **CSP (Content Security Policy)**:
   - Check console for CSP errors
   - View > Developer > JavaScript Console
   - Look for "refused to execute" messages

3. **JavaScript Conflicts**:
   - Check if SCORM content overrides window.API
   - Check if SCORM content blocks parent window access

4. **Session Timeout**:
   - User might be logged out
   - Check Authentication in Network tab

MANUAL FIX (if automated doesn't work):
----------------------------------------

If Save & Exit button doesn't work, user can:
1. Close the browser tab directly (data auto-saves)
2. Use browser back button
3. Navigate away - beforeunload handler will save data

The system has automatic save on:
- Page unload
- 30-second intervals (heartbeat)
- Visibility change (tab switch)
- SCORM Commit/Terminate calls
